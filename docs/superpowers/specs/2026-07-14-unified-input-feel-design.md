# ビートソード 統一入力・演出強化設計

日付: 2026-07-14
対象ファイル: `game.js`, `index.html`, `style.css`

## 背景

実プレイフィードバックに基づく追加要望。防御ノーツの予告不足、攻撃/能力/防御のキー分けによる操作の複雑さ、リズムの伝わりにくさ（視覚・聴覚フィードバック不足）への対応。

## スコープ

先行実装（ウェーブ制・6キャラクター・R/O防御システム・ノーツ予測表示・小節アライン等）はすべて維持する。

## 1. 防御ノーツの予告時間延長＋左から流す

`Enemy.startAttack()`の`attackTimer`を1.0秒→2.5秒に延長し、`attackWarning`を攻撃開始直後（`attackTimer < 2.5`、実質即時）から立てる。防御ノーツ(`type: 'defend'`)は、Canvas描画時のx座標計算を他のノーツと逆方向にし、画面左から中央の判定ラインへ流れるようにする（攻撃・能力ノーツは引き続き右から）。

## 2. ノーツアイコンの簡略化

キー別入力を廃止するため、ノーツに`R`/`O`/`R+O`の文字ラベルは表示しない。色・形（剣=オレンジ円、能力=青円、防御=赤ひし形）のみで種類を区別する。

## 3. 統一入力（どのキー／タップでも直近のノーツを自動判定）

`RhythmSystem`に`checkInputAny()`を追加し、剣・能力・防御の3プールを横断して現在時刻に最も近い未解決ノーツを探し、そのノーツの種類に応じて既存の`checkInput(type)`ロジックへ委譲する。`GameController`はR/O個別のキーバインド・コンボ検出・専用タップボタンを廃止し、「任意のキー押下」または「画面（ゲーム領域）のどこかをタップ/クリック」で共通の`handleUniversalInput()`を呼ぶ。剣ヒット時のダメージ処理は`resolveSwordHit(result)`として切り出す。

## 4. 効果音の強化

- ノーツ成功時（Perfect/Great/Good、種類問わず）に共通の成功効果音を追加で鳴らす
- 毎拍「タン」という短いクリック音を追加し、リズムを聴覚的に分かりやすくする（`onBeat`コールバックで再生）

## 5. 拍連動の縦揺れ演出

プレイヤー・敵の描画位置に、現在拍（`audio.getCurrentBeat()`）に連動した小さな縦方向のオフセットを加え、キャラクターがリズムに合わせて弾むように見せる。

## 今回のスコープ外（変更しない）

- ウェーブ制・敵AI・成長システム・BGM実音源・キャラ別攻撃/能力VFXの中身は変更しない
- 判定式（Perfect/Great/Good/Miss）自体は変更しない
