# ビートソード ゲームプレイ刷新設計

日付: 2026-07-12
対象ファイル: `game.js` / `index.html` / `style.css`

## 背景

`docs/superpowers/specs/2026-07-12-beatsword-fix-design.md` の修正（Task 1〜8、以下「先行修正」）を実装し、実際にブラウザで動作確認したところ、ユーザーから以下のフィードバックを得た。これはバグ修正ではなく、コアゲームプレイの方向性そのものに関わる変更である。

- 強制横スクロールをやめたい
- 剣攻撃のノーツは、プレイヤーが攻撃を選んだ（Zを押した）ときだけ発生してほしい
- カウンター要素を廃止したい
- 敵をもっと増やし、一斉になぎ倒す爽快感を出したい
- リズム判定をもう少し緩めたい
- UIの重なりを直したい

## スコープ

先行修正（Task 1〜8）で実装したBGM実音源化・画像アセット・敵タイプ追加・固有能力個別実装・キャラ削除はすべて維持する。今回はその上に、ステージ構造・入力方式・敵配置・UIレイアウトを変更する。

## 1. 横スクロール廃止 → ウェーブ制の固定アリーナ

### 現状
`StageManager`が毎フレーム`scrollX`を加算し続け、画面が自動で右へ流れる。敵は画面右外から出現し、プレイヤーの後方（`scrollX + 50`より手前）には戻れない。

### 変更後
- `scrollX`の自動加算を廃止し、常に`0`固定とする（＝背景・敵・プレイヤーの座標系がそのまま画面座標になる）
- プレイヤーは画面内（例: `x: 50 〜 CANVAS_WIDTH - 50`）を自由に左右移動できる。既存の後方クランプ処理（`scrollX`基準の位置制限）は不要になるため削除する
- 敵は画面の**左端**または**右端**からランダムに出現し、プレイヤーに向かって歩いてくる（既存の「最も近いプレイヤーへ接近する」ロジックはそのまま使えるが、出現位置ロジックを画面右外一辺倒から左右両端に変更する）
- ステージは「ウェーブ」の連続で構成する。1ウェーブ＝一定数の敵の同時出現。ウェーブ内の全敵を倒すと次のウェーブが一定間隔後に出現する
- ステージクリア条件: 先行修正で導入した「BGMの長さに応じたステージ長」の考え方を踏襲し、**選択されたBGMトラックの再生時間から総ウェーブ数を算出**する（例: `totalWaves = floor(trackDurationSeconds / waveIntervalSeconds)`、`waveIntervalSeconds`は目安15秒）。全ウェーブを出現させ切り、かつ画面上に敵が残っていない状態になったらステージクリアとする
- エリート出現（現行「距離85%到達時」）は「最終ウェーブ到達時」に読み替える

## 2. ノーツはZ押下時にバースト発生

### 現状
`Game.startGame()`内の`onBeat`コールバックが、BGM再生中は常時・自動的に剣攻撃ノーツ(`generateSwordNotes`)を生成し続けている。

### 変更後
- `onBeat`コールバックによる自動ノーツ生成を廃止する
- 既存の能力(X)と同じ方式に統一する: **Zキーを押した瞬間に、その場から数拍分（既存の能力と同程度、目安4拍）の剣攻撃ノーツ列を生成**する。生成中は追加でZを押しても新しい列は開始しない（既存ノーツ列が終わるまで待つ）
- ノーツ列の入力が終わった後、再度Zを押せば新しい列が始まる。これにより「何もしなければノーツは流れてこない」という要求を満たす
- `RhythmSystem`には既に`startAbility(beats)`という「特定拍数だけノーツ列を生成する」仕組みがあるため、剣攻撃用に同種のメソッド（例: `startSwordBurst(beats)`）を新設し、`RhythmSystem.notes`のうち`type: 'sword'`のノーツをバースト方式に切り替える

## 3. カウンター機能を全廃

以下を削除する。
- `RhythmSystem`: `generateCounterNote`、ノーツタイプ`'counter'`関連の生成・判定・描画コード
- `Enemy`: `attackWarning`・`counterable`フィールドとその切り替えロジック、`resolveCounter`メソッド。`executeAttack`内の「カウンター中なら無効化」分岐を削除し、衝突時は常に`p.takeDamage(this.atk)`（接触ダメージ、先行修正で「現状維持」と決めた方式）のみになる
- `Player`: `isCountering`・`counterTimer`・`counter()`メソッド、カウンター用の見た目演出（赤いリングエフェクト）
- `Game`: `handleCounter`、`counterWindow`フィールド、Cキーの入力バインディング
- `Renderer`: 敵の`COUNTER!`表示、カウンター判定用のノーツ描画（ひし形アイコン）
- `index.html`: 「遊び方」画面・チュートリアルボックスのカウンター説明文言、Cキーのヒント表示
- `tests/counter-fix.test.js`: 機能ごと削除する（`Enemy.resolveCounter`が存在しなくなるため）

**維持するもの**: 自爆敵・ヒーラー敵が使っている攻撃タイマー（`isAttacking`/`attackTimer`/`startAttack`/`executeAttack`の骨格）は、カウンターとは独立した「行動タイミング管理」として引き続き使用する。

## 4. 敵の大量出現＋ノックバック演出

- 1ウェーブあたりの同時出現数を増やす（現行の1〜3体から、目安5〜10体程度。ステージが進むほどさらに増える）
- 剣攻撃・能力がヒットした敵は、即座に消滅させるのではなく「ヒットした方向と逆に高速で吹き飛びながら縮小・透明化して消える」演出を`Enemy`に追加する（例: `knockbackVx`/`knockbackTimer`のような一時状態を持たせ、`Renderer.renderEnemy`側でその状態のときは通常の敵描画の代わりに吹き飛び演出を描画する）。当たり判定への影響はなく、あくまで見た目の演出のみ

## 5. 判定を緩和

`CONSTANTS`の判定ウィンドウを以下の値に変更する。

| 定数 | 現状 | 変更後 |
|---|---|---|
| `PERFECT_WINDOW` | 0.06 | 0.09 |
| `GREAT_WINDOW` | 0.12 | 0.18 |
| `GOOD_WINDOW` | 0.20 | 0.30 |

## 6. UI重なりの修正

`#comboDisplay`・`#beatBar`・`.tutorial-box`が、いずれも画面下部に固定px（`bottom: 140px`・`bottom: 20px`・`bottom: 120px`など）で個別に絶対配置されており、`#gameContainer`が縮小されると（`aspect-ratio`維持のまま幅が狭くなる場面）重なって見える。これらを1つの縦積みラッパー（例: `#bottomHud`）にまとめ、`position: absolute; bottom: 0;`＋`display: flex; flex-direction: column;`で内部的に積み上げる形に変更し、個々の`bottom`固定値指定を廃止する。これにより画面サイズによらず重ならずに積み上がる。

## 今回のスコープ外（変更しない）

- BGM実音源・画像アセット・敵タイプ（自爆敵・ヒーラー敵）・キャラ別固有能力・6キャラクター構成は先行修正のまま維持
- LANマルチプレイの実通信は引き続き未実装
