# ビートソード 敵視認性・演出強化・BGMループ修正設計

日付: 2026-07-15
対象ファイル: `game.js`, BGM音声ファイル4点(`79拍:分1.mp3`, `79拍:分2.mp3`, `90拍:分.mp3`, `110拍:分.mp3`)

## 背景

統一入力・演出強化計画(unified-input-feel)完了後の実プレイフィードバック。50体規模のウェーブで敵が重なって見づらい、全体的な躍動感が弱い、BGMのループ継ぎ目が不自然、という指摘への対応。

## スコープ

先行実装(ウェーブ制・6キャラクター・統一入力・防御ノーツ左流し・拍連動縦揺れ等)はすべて維持する。

## 1. 敵のウェーブ出現を分散させる

`StageManager.spawnWave()`で生成する各`Enemy`に、生成時点でランダムな`spawnDelay`(0〜3秒)を持たせる。`Enemy.update(dt, ...)`は`spawnDelay > 0`の間、`spawnDelay -= dt`するのみで、移動・攻撃・プレイヤーとの当たり判定などの通常ロジックは一切実行しない。`Renderer.renderEnemy`は`spawnDelay > 0`の敵を描画しない(未出現として扱う)。

あわせて、`spawnWave()`内で敵のy座標を`CONSTANTS.GROUND_Y`を中心に`±15px`のランダムなオフセットで生成する(`y = CONSTANTS.GROUND_Y + (Math.random() - 0.5) * 30`)。既存の当たり判定ボックス(`getHitbox()`/`getAttackHitbox()`)は敵の種類ごとに高さ40〜90pxあり、±15pxのオフセットでは判定の取りこぼしは発生しない。

ウェーブクリア判定(`this.enemies.length === 0`)は`spawnDelay`に関わらず配列に存在する全敵が対象のままとし、変更しない(未出現の敵も含めて全滅するまでウェーブは継続する)。

## 2. パーフェクト時のプレイヤー演出(ダッシュ+発光)

`GameController.handleUniversalInput()`で、剣ノーツの判定が`perfect`だった場合、`Player`に短時間の前方ダッシュ状態を発生させる。`Player`に`dashTimer`(秒)を追加し、非ゼロの間は`Player.update()`で移動先方向に通常より速いスピードで数フレーム前進させてから停止する(「シュバッと動いてピシッと止まる」)。`Renderer.renderPlayer`は`dashTimer > 0`の間、プレイヤーに発光(グロー)エフェクトを追加描画する。

被弾した敵側には新規の演出を追加しない(既存のノックバック処理のみ)。

## 3. BGM音声ファイルのシームレスループ修正

現在`AudioSystem.startBGM()`は`source.loop = true`のみでフェード処理は行っておらず、ループの継ぎ目の不自然さは音声ファイル自体の先頭/末尾の無音パディングに起因すると判明した(ffmpegの`silencedetect`で確認済み)。

対象ファイルをffmpegで直接トリミングし、上書きする(コード変更なし):

| ファイル | 先頭トリム | 末尾トリム(トリム後の終端) |
|---|---|---|
| `110拍:分.mp3` | 0.53s | 111.13s |
| `79拍:分1.mp3` | 0.53s | 47.29s |
| `79拍:分2.mp3` | 0.53s | 126.18s |
| `90拍:分.mp3` | 0.53s | 90.24s |

`90拍:分.mp3`の曲中37秒付近の無音区間は楽曲自体の一部(ブレイク)であり対象外とする。トリミングは元の音質(ビットレート)を維持して再エンコードする。

## 4. 防御/回避/カウンター入力統一の確認

統一入力計画(unified-input-feel)のTask 3で`RhythmSystem.checkInputAny()`により既に実装済み。専用キーバインド・専用タップボタンは`game.js`/`index.html`/`style.css`のいずれにも残存していないことをgrepで確認済み。本ラウンドでのコード変更は不要。最終手動検証チェックリストに項目として含める。

## 今回のスコープ外(変更しない)

- ウェーブ制の全体的な進行ロジック(`totalWaves`計算等)
- 判定式(Perfect/Great/Good/Miss)自体
- 敵の種類別AI・ダメージ計算
- BGM選曲ロジック(`pickRandomTrack`)
